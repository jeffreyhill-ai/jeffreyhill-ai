## Current Focus

I am building the public proof trail around **MoneyVision**, a local-first AI advisor workbench and agent harness for wealth services.

MoneyVision is not meant to be just another static dashboard. It is an advisor operating layer upstream of dashboards, planning tools, CRMs, spreadsheet models, and client portals:

```text
messy evidence
  -> AI-assisted extraction
  -> source-linked facts
  -> advisor review tasks
  -> approved financial model
  -> client story, display plan, snapshot, or sanitized export
```

The product thesis: advisors should be able to use AI to normalize messy client evidence, make uncertainty visible, approve what matters, and send only reviewed and sanitized data to the systems they already trust.

## The Thesis

The next generation of wealthtech will not be won by prettier dashboards alone.

It will be won by systems that can:

- turn messy documents, statements, screenshots, spreadsheets, notes, and provider feeds into structured, source-linked facts
- keep uncertainty visible instead of hiding it inside polished prose
- route low-confidence facts, missing evidence, contradictions, assumptions, and narratives through advisor review
- connect AI reasoning to deterministic tools and auditable state changes
- preserve entitlements, attribution, lineage, and point-in-time context
- prepare reviewed data for downstream systems without exposing raw client documents unnecessarily

That is the space I am building in: financial AI that behaves less like a chatbot and more like a trusted advisor operating layer.
